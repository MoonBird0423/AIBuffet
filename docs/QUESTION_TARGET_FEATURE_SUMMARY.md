# 提问对象功能实施总结

## 功能概述
为聊天系统添加了提问对象功能，用户可以针对特定的图书或知识库进行对话，系统会在数据库中持久化保存这些关联关系。

## 核心特性
1. **提问对象选择**: 支持选择图书或知识库作为提问对象
2. **状态持久化**: 提问对象信息保存在数据库中
3. **会话恢复**: 重新打开会话时自动恢复提问对象
4. **动态管理**: 支持在会话中清除提问对象
5. **视觉提示**: 清晰的UI显示当前提问对象和操作提示

## 数据库变更

### 新增迁移文件
- `V13__add_question_target_to_chat_sessions.sql`
  - 添加 `question_target_type` 字段：提问对象类型（book/knowledge）
  - 添加 `question_target_id` 字段：提问对象ID
  - 添加 `question_target_name` 字段：提问对象名称
  - 添加索引优化查询性能

### 模型更新
- `ChatSession` 实体类添加了新字段支持

## 后端API变更

### 新增DTO类
1. **CreateChatRequest**
   - 支持创建会话时传递提问对象信息
   
2. **QuestionTargetRequest**
   - 用于更新提问对象的专用DTO

### 服务层更新
1. **ChatService接口扩展**
   - `createChatSession()` 方法重载，支持提问对象参数
   - `updateQuestionTarget()` 更新提问对象
   - `clearQuestionTarget()` 清除提问对象

2. **ChatServiceImpl实现**
   - 完整实现所有新接口方法
   - 支持提问对象的CRUD操作

### 控制器层更新
1. **ChatController扩展**
   - 更新创建会话接口支持提问对象
   - 新增 `PUT /api/chats/{sessionId}/question-target` 接口
   - 新增 `DELETE /api/chats/{sessionId}/question-target` 接口

## 前端功能实现

### API层更新
1. **api.js扩展**
   - `createChatSession()` 支持可选的 questionTarget 参数
   - `updateQuestionTarget()` 新增API函数
   - `clearQuestionTarget()` 新增API函数

### 页面逻辑更新
1. **Chat.js核心逻辑**
   - URL参数解析：支持 bookId/bookName 和 knowledgeBaseId/knowledgeBaseName
   - 会话恢复：从数据库恢复提问对象状态
   - 创建会话：传递提问对象信息到后端
   - 状态管理：动态更新和清除提问对象

2. **ChatHeader组件**
   - 显示当前提问对象信息
   - 提供删除提问对象的操作
   - 显示未设置提问对象的提示

## 用户流程

### 场景1：从图书详情页进入聊天
1. 用户在图书详情页点击"AI对话"
2. 系统跳转到聊天页面，URL包含图书信息
3. 聊天界面显示针对该图书的提问对象标签
4. 用户发送消息时，系统创建带有图书关联的会话

### 场景2：从知识库进入聊天
1. 用户在知识库页面点击"开始对话"
2. 系统跳转到聊天页面，URL包含知识库信息
3. 界面显示针对该知识库的提问对象标签
4. 会话创建时关联到该知识库

### 场景3：恢复已有会话
1. 用户打开历史会话
2. 系统从数据库读取会话的提问对象信息
3. 界面自动恢复显示提问对象标签
4. 保持会话的上下文连续性

### 场景4：清除提问对象
1. 用户点击提问对象标签的删除按钮
2. 系统调用API清除数据库中的关联关系
3. 界面更新为通用聊天状态
4. 显示设置提问对象的提示

## 技术实现要点

### 数据持久化
- 在创建会话时保存提问对象信息
- 支持后续更新和清除操作
- 数据库索引优化查询性能

### 状态管理
- 前端使用React状态管理提问对象
- URL参数优先级高于数据库状态
- 支持动态状态切换

### API设计
- RESTful风格的API设计
- 支持创建、更新、删除操作
- 错误处理和容错机制

### 用户体验
- 清晰的视觉提示
- 一致的交互体验
- 渐进式功能引导
