## 概述

AIBuffet 采用基于 JWT 的身份认证系统，结合 Spring Security 进行权限控制。系统设计允许未登录用户访问公开的图书内容，同时保护需要认证的功能和私有内容。

### 核心原则

1. __公开图书馆__：已发布的图书内容对所有用户开放
2. __私有内容保护__：未发布的内容只有创建者可以访问
3. __功能分级__：基础浏览功能无需登录，高级功能需要认证

## 权限控制实现

### Spring Security 配置

在 `SecurityConfig.java` 中定义了无需认证的路径：

```java
.requestMatchers("/api/auth/**").permitAll()
.requestMatchers("/api/knowledge-bases/public/**").permitAll()
// 无需登录的图书功能
.requestMatchers(HttpMethod.GET, "/api/documents").permitAll()                    // 图书搜索、分类筛选
.requestMatchers(HttpMethod.GET, "/api/documents/*").permitAll()                 // 图书信息查询  
.requestMatchers(HttpMethod.GET, "/api/publish/docs/*/interpretation").permitAll() // 图书文字解读查询
.requestMatchers(HttpMethod.GET, "/api/documents/*/audio").permitAll()           // 获取音频URL
.requestMatchers(HttpMethod.GET, "/api/documents/*/audio/status").permitAll()    // 音频状态查询
.anyRequest().authenticated()
```

### 认证过滤器

`AuthenticationFilter.java` 实现了以下逻辑：

1. 尝试解析 JWT token（如果存在）
2. 不强制要求所有请求都有 token
3. 让 Spring Security 决定是否需要认证

### 业务层权限控制

#### 文档访问权限

在 `DocumentService` 中实现：

- __已发布文档__：所有用户都可以访问
- __未发布文档__：只有文档创建者可以访问

```java
// 检查文档发布状态和用户权限
if (docFile.getPublishStatus() != DocFile.PublishStatus.PUBLISHED) {
    if (userId == null) {
        throw new IllegalArgumentException("No permission to access this document");
    }
    // 验证用户是否为文档创建者
}
```

#### 音频访问权限

在 `AudioSynthesisService` 中实现安全方法：

- `getAudioUrlSecure()`：带权限检查的音频URL获取
- `hasAudioSecure()`：带权限检查的音频状态检查

## 安全策略

### 1. 文档发布状态控制

- __PUBLISHED__：公开可访问
- __UNPUBLISHED__：仅创建者可访问

### 2. 用户权限验证

- 通过 `documentService.getDocument(docId, userId)` 进行统一权限检查
- 支持 `userId = null` 的未登录用户场景

### 3. 错误处理

- 无权限访问返回权限错误，不暴露资源是否存在
- 统一的错误响应格式

### 4. Token 处理

- 可选的 token 认证（`@AuthenticationPrincipal(errorOnInvalidType = false)`）
- 无效 token 不影响公开接口访问

## 技术实现细节

### Controller 层实现

#### 可选认证参数

```java
@GetMapping("/{docId}/audio")
public ResponseEntity<ApiResponse<Map<String, Object>>> getAudioUrl(
        @PathVariable Long docId,
        @AuthenticationPrincipal(errorOnInvalidType = false) User user) {
    Long userId = user != null ? user.getId() : null;
    // 使用安全方法进行权限检查
    String audioUrl = audioSynthesisService.getAudioUrlSecure(docId, userId);
}
```

#### 必需认证参数

```java
@PostMapping("/{docId}/audio/synthesize")
public ResponseEntity<ApiResponse<Map<String, Object>>> synthesizeAudio(
        @PathVariable Long docId,
        @AuthenticationPrincipal User user) {
    // user 不能为 null，Spring Security 会在此之前拦截
}
```

### Service 层实现

#### 权限检查方法

```java
public String getAudioUrlSecure(Long docId, Long userId) {
    try {
        // 通过 DocumentService 进行权限检查
        documentService.getDocument(docId, userId);
        return getAudioUrl(docId);
    } catch (Exception e) {
        // 记录日志但不抛出异常，返回 null
        return null;
    }
}
```

## 维护说明

### 添加新的公开接口

1. 在 `SecurityConfig.java` 中添加 `permitAll()` 配置
2. 在 Controller 中使用 `@AuthenticationPrincipal(errorOnInvalidType = false)`
3. 在 Service 层实现相应的权限检查逻辑

### 添加新的认证接口

1. 确保 `SecurityConfig.java` 中没有对应的 `permitAll()` 配置
2. 在 Controller 中使用 `@AuthenticationPrincipal User user`
3. 在 Service 层假设 userId 不为 null

### 测试权限控制

1. __未登录用户测试__：测试公开接口的访问
2. __已登录用户测试__：测试完整功能访问
3. __权限边界测试__：测试用户只能访问自己的资源
