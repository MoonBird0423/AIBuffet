# 当前提示词配置

本文档记录了系统中现有的三个提示词模板，这些提示词将从配置文件迁移到数据库进行版本管理。

## 1. 图书解读提示词

**名称**: `book.interpretation.user-prompt`  
**版本**: v1  
**描述**: 用于生成图书解读对话的提示词，生成多角色参与的图书分享会形式的内容  

**内容**:
```
请根据上传图书生成一段对话

场景：​​ 一场多角色参与的图书分享会。

角色：
-主持人：年轻女性，负责开场、结束致辞，对嘉宾提问。
-嘉宾：中年男性，负责解读图书，回答主持人提问。
-观众代表：中年女性，负责分享感悟。

输出模板：
### 主持人:
【（欢迎语）欢迎大家来到书意平台的图书分享会，我是主持人默默。】
【
- （开场三选一）  
  - （悬念痛点）比如："你所相信的（什么）真理，可能全是虚构——这本书彻底颠覆了我的认知"
  - （金句冲击）列举书中的金句 + "这仅仅是开始..."  
  -（ 场景共鸣）比如："深夜焦虑（什么）问题？作者用亲身经历给出了震撼答案"
】
【 "从本书你将获得：
① (核心思想X个 )···
②( 独家解法 )···
③ （现实启发）···
】
【（过渡语）比如：今天我们有幸邀请到了本书的作者/译者/编辑（名字），接下来···】
### 嘉宾：
【分享写作动机、时代背景、领域发展现状，例：《人类简史》诞生于全球化与科技爆炸的反思期。】
​​【本书试图回答xxx问题？（工具书：解决什么痛点？小说：探讨什么主题？）】
### 主持人：
【（分享感悟）比如：针对（什么）社会现象，这本书将发挥非常大的作用···】
---
### 主持人：
【（主持人核心观点提问1）能聊一聊本书的第核心观点（xx具体观点）吗？···】

- "举个书中案例或故事：..."(选最具象的1个进行详细描述，用讲故事的手法包含起承转合。)
】
### 主持人：
【（分享感悟）"这能帮解决现实中的XX困境···" (必须关联痛点，可列举现象、分析本质、描述观点如何发挥作用)】
---
### 主持人：
【（主持人核心观点提问2）比如：能聊一聊本书的核心观点（xx具体观点）吗？···】
### 嘉宾：
【 回答内容结构同上】
### 主持人：
【分享感悟结构同上】

### 嘉宾：
【 回答内容结构同上】
### 主持人：
【分享感悟结构同上】
---
### 主持人：
【（主持人核心观点提问4）比如：能聊一聊本书的核心观点（xx具体观点）吗？···】
### 嘉宾：
【 回答内容结构同上】
### 主持人：
【分享感悟结构同上】
---
### 主持人：
【（主持人核心观点提问5）比如：能聊一聊本书的核心观点（xx具体观点）吗？···】
### 嘉宾：
【 回答内容结构同上】
### 主持人：

### 嘉宾：
【
据书籍类型选择创新点
| 类型       | 剖析方向                  | 关键动作               |
|------------|------------------|------------------------|
| 工具书 | 亲测方法论有效性       | 曝实践效果+踩坑经验    |
| 小说   | 角色复杂性/隐喻解析     | 撕开人性矛盾层         |
| 社科   | 颠覆性争议点          | 摆正反观点+你的立场    |

### 嘉宾：
【从个人/社会两个方面来总结本书的价值，（个人价值）比如技能提升、认知刷新、情感共鸣等，请详细展开。（社会​​价值）比如揭示问题、推动讨论、提供解决方案，请详细展开。】
【书中未解决的矛盾或学界批评。保持批判性​​：即使是经典，也可指出其时代局限。】
---
### 主持人：
【(结束致辞)比如：感谢大家参加这次分享会，我们共同经历了非常美妙的时刻，相信各位听众也和我一样收获颇多，····】
【
（推荐图书）  
  -（适合人群）比如：正在为孩子的教育而困扰的家长，尤其是那些希望用更科学、更人性化的教育方法来引导孩子的父母。
  -（必读理由）比如：书中提供的方法和工具不仅实用，而且能够从根本上改变亲子关系，让孩子在充满爱与尊重的环境中健康成长。
】
【（给读者的话）比如：可引用名人名言/名人事迹对书籍主题进行升华，鼓励用户进行实践，让解读更具备感染力。】

​​特别注意：​​ 
-不要直接输出"【】"符号以及其中的提示内容，根据图书内容进行总结替换。
-输出对话8-12轮，除了对话不要输出任何其他多余的内容或提示。
-输出内容使用口语化，生活化的语言。
-输出不少于3000tokens。
```

## 2. 思维导图提示词

**名称**: `book.mindmap.user-prompt`  
**版本**: v1  
**描述**: 用于生成图书思维导图的提示词，支持多种图书类型的结构化分析  

**内容**:
```
你是一个脑图专家。请解析用户上传的书籍内容生成脑图，按下面的流程进行解析：
一、类型识别
判定为以下类型之一：
[工具类｜小说类｜人文社科类｜传记与纪实类｜科普类｜学术类]

二、信息提取
1. 基础信息（必填）：
- 书籍名称（精确提取）
- 作者（含国籍，如：马尔克斯（哥伦比亚））
- 主类型 & 子类型（如：工具类→时间管理；小说类→科幻小说）
-书籍主题（一句话深度总结)
2. 章节解析（确保覆盖所有章节内容，如果章节内容过多可将多个章节按文章逻辑合成一个章节）：
[章节名称] 
├─ 核心观点（每个观点独立节点，论点支持多层级嵌套，一个观点的子观点、案例、结论都放成这个观点的子节点）
├─ 支撑案例（含数据/实验/故事）
└─ 推导结论（说明与论点的逻辑关系）
3. 关键信息（根据文章内容和类型梳理关键信息，至少选2个内容项进行梳理）：
■ 工具类：
- 待解决问题（3-5个痛点）
- 方法论体系（步骤流程）
- 应用工具（清单化呈现）
■ 小说类：
- 角色档案（姓名+核心特征）
- 关系图谱（可视化箭头连接）
- 情节里程碑（含关键转折点）
■ 传记类：
- 人生坐标轴（时间线标注重大事件，时间：事件）
- 成就清单（量化呈现）
- 关键抉择点（影响命运的选择）
■ 科普类：
- 核心概念树（专业术语层级化）
- 实验/现象档案（原理+意义）
- 现实应用场景（案例说明）

三、请按照下面的格式要求输出前面提取的基础信息、章节信息、扩展信息，层级结构分支数量根据提取的信息进行动态调整，格式要求为markdown。
# 书名
## 基础信息
### 作者
- **姓名**: xxxxx
- **国籍**: xxx
### 分类
- **主分类**: xxx
- **子分类**: xx
### 书籍主题
- **主题**: xxxx
## 章节解析
### 章节名称
#### xx观点xx
- **xx子观点xx**：xx说明xx
- **xx案例xx**：xx说明xx
- **xx结论xx**：xx说明xx
## 关键信息
### xxxx信息名称，比如待解决问题xxx
- **xx说明xx**

特别注意：
-类型识别和信息提取过程不用输出，直接输出结构化markdown就行了，切记保证md格式的准确性。
-严格参照上传内容，确保输出的真实性和完整性。
```

## 3. 测试题提示词

**名称**: `book.quiz.user-prompt`  
**版本**: v1  
**描述**: 用于生成图书测试题的提示词，围绕核心观点生成选择题  

**内容**:
```
你作为一个出题专家，请根据文档出10道题目。
1、信息提取：请先从文档中提取10个核心观点，和支撑该观点的案例、推导出的结论；
2、出题：围绕每个观点出一道选择题，题目以提取的观点来编写一个案例让用户做出选择，每道题4个选项，只有一个选项是符合作者核心观点的。
3、解析：给出正确答案选项，并解释该选项为什么是正确的。并给出其他选项错误的原因。
4、结构化输出markdown格式如下：
### 001. [题干]
**选项：** 
A. 内容 
B. 内容 
C. 内容 
D. 内容 
**答案：** 
[字母] 
**解析：** 
✓ 正确项说明（含观点关联分析）
✗ 错误项原因（每个选项1句，引用文档矛盾点） 

特别注意：
1、信息提取过程不用输出，直接输出结构化markdown就行了，切记保证md格式的准确性。
2、严格参照上传内容，确保输出的真实性和完整性。
```

## 4. 音频SSML生成提示词

**名称**: `audio.ssml.generation.prompt`  
**版本**: v1  
**描述**: 用于将对话内容转换为阿里云CosyVoice大模型的SSML标记语言的提示词  

**内容**:
```
请将下面的对话转换成阿里云CosyVoice大模型的SSML标记语言（基于W3C的语音合成标记语言版本1.0）让对话的每个角色都有不同的音色。
可使用标签：
1、<speak>标签是所有待支持SSML标签的根节点。一切需要调用SSML标签的文本都要包含在<speak></speak>中；
2、支持多个 <speak> 标签并列使用（如：<speak></speak><speak></speak>），但不支持嵌套结构（如：<speak><speak></speak></speak>）；
3、<break>用于在文本中插入停顿，该标签是可选标签；
4、<phoneme>用于控制标签内文本的读音，该标签是可选标签；
5、<say-as>用于指示出标签内文本的信息类型，进而按照该类型的默认发音方式发音；
使用例子：
<speak><say-as interpret-as="telephone">114</say-as>查询号码 <say-as interpret-as="cardinal">123</say-as>开始干。加起来为<say-as interpret-as="digits">1234</say-as>。<say-as interpret-as="name">张三</say-as>的快递。<say-as interpret-as="address">富路国际1号楼3单元304</say-as><say-as interpret-as="nick">李四6689</say-as></speak>
角色音色选择指南：（通过<speak>标签属性voice来指定）
-longxiaoxia_v2：适合主持人
-longcheng_v2：适合会议男嘉宾
-longxiaochun_v2：适合会议女嘉宾
-longwan_v2：适合观众代表
特别注意：
-除了转换内容不要输出其他内容，需要保证输出内容可直接CosyVoice大模型识别。
-需要对输出内容进行检查，确保格式满足规范要求。
-对话流畅，信息完整。
对话内容
%s
```
